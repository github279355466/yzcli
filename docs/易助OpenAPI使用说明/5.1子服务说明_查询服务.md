# 查询服务

> 所属章节: 子服务说明

---

用于频繁查询资料或单据列表。

由于query查询子服务，是通过TPAOpenAPIProcess.exe每次调用易助dll执行任务并通过文件形式传递数据，对服务稳定性和服务器性能有比较大的压力，不适用于频繁的定时查询。

fastquery查询子服务，获取到SQL进行缓存，之后根据令牌不同生成最终查询语句，直接查询数据库，无需每次调用易助DLL，缺点是返回内容无法进行内含外显处理。

服务接口名称命名规则：yz.oapi.{Typekey}.fastquery

#### 输入字段

| 标签 | 说明 |
| --- | --- |
| load_data_browse_config | 加载数据自定义设计数据浏览界面配置，默认为false 如果为true，则字段顺序，字段是否显示都会和自定义设计一致 |
| pageSize | 页面尺寸（每页数据量），最大10000 |
| pageNo | 页码，从1开始。 |
| use_has_next | 是否分页 SQL Server 2012及以上版本  支持OFFSETc 和 FETCHc 关键字语法，是用于分页查询的功能，为真分页查询 SQL Server 2008R2及以下版本为查询所有数据后，定位到所需数据，再返回，为假分页查询 |
| conditions | 条件集合，用于对查询结果进行二次筛选。 |
| conditions.operator | 条件操作符：and（默认值）、or。
如果操作符为and，此Tag可缺省。 |
| conditions.group | 多组条件集合，有多组合条件时使用
如：group1(条件1 and 条件2) or  group2(条件3 and 条件4) |
| conditions.fields | 条件数组 |
| conditions.fields.field_name | 条件字段名称（可使用'+'号拼接字段） |
| conditions.fields.value | 条件字段值 |
| conditions.fields.operator | 逻辑操作符：>、>= 、=、 <、 <= 、like
IN、NOT IN 、BETWEEN、EXISTS、NOT EXISTS                        
如果操作符为=，此Tag可缺省。 |
| orders | 排序数组 |
| orders.field_name | 排序字段名称 |
| orders.order_type | 排序类型：asc  desc |
| query_type | all ：返回所有字段 key：只返回key字段 |

#### 返回字段

| 字段 | 说明 |
| --- | --- |
| total_result | 满足条件的数据总笔数 |
| page_count | 总页数 |
| page_no | 当前页码 |
| has_next | 是否有下一页 |
| cnt | 本页数据笔数 |

#### 示例场景

获取客户以 （KH0002开头 或者 币种是RMB）或者（部门在0201、02中  且  经办人编号是060601）的订单列表。

结果集按客户编号正序排序。

进行分页，每页10笔数据，查询第二页，即第11笔到第20笔数据

# 查询条件使用說明

## OPENAPI 查询条件 方法说明

概述说明:

OPENAPI Query接口，其查询条件可支持多种方式，如下进行说明。

Query Request格式：

{
  "std_data": {
    "parameter": {
      "page_no": 1,
      "page_size": 10,
      "conditions": [
        {
          "groups": [
            {
              "fields": []
            }
          ]
        }
      ],
      "orders": []
    }
  }
}

page_size:为期望回传数据每页数据笔数。

page_no:为期望取回第n页资料。

conditions:为查询的条件组，包含以下二个部分，除了要取得全部资料以外，皆为必传。

groups为查询条件组，用于组合多个fields。conditions与fields中间层级节点名称不固定，只需要遵循XXXX，XXXX_operator的规则，例如：

{
  "std_data": {
    "parameter": {
      "page_no": 1,
      "page_size": 10,
      "conditions": [
        {
          "WWWW": [
            {
              "fields": [],
              "XXXX": [
                {
                  "fields": []
                }
              ],
              "XXXX_operator": "AND"
            }
          ],
          "WWWW_operator": "AND"
        },
        {
          "YYYY": [
            {
              "fields": [],
              "XXXX": [
                {
                  "fields": []
                }
              ],
              "XXXX_operator": "AND"
            }
          ],
          "YYYY_operator": "OR"
        }
      ],
      "orders": []
    }
  }
}

XXXX_operator允许为空，默认为AND

fields为嵌套末级节点，是一个字段条件，可以为单一字段条件，亦可以是多字段复合条件组合。

operator为fields中各个条件组之间的关系，可以为AND、OR。

orders为SQL排序条件，请参考最后一点。

名词说明:

以下为范例说明

### 取得数据表全部资料

conditions请传入一个空的JSON Object。

"conditions": []

### 单一条件式

SELECT * FROM TPADEA

WHERE DEA001='001'

{
  "std_data": {
    "parameter": {
      "page_no": 1,
      "use_has_next": true,
      "page_size": 10,
      "conditions": [
        {
          "groups": [
            {
              "fields": [
                {
                  "field_name": "item_no",
                  "operator": "=",
                  "value": "001"
                }
              ]
            }
          ]
        }
      ]
    }
  }
}

在这个范例中，WHERE仅有一个字段，因此仅需一个条件式即可满足。

field_name填入字段节点名。operator为比较运算符，可以为<,<=,=,>=,>,<>,LIKE,BETWEEN,IN,NOT IN,EXISTS,NOT EXISTS。

value为条件值，字符串使用双引号"包裹；数值请勿使用双引号。

因为fields仅单一条件式，因此operator并无作用。但仍然需要传入。

### 多个条件式

SELECT *  FROM DCSIBA

WHERE IBA001='20250515001102' AND IBA003='20090515'

{
  "std_data": {
    "parameter": {
      "page_no": 1,
      "use_has_next": true,
      "page_size": 10,
      "conditions": [
        {
          "groups": [
            {
              "fields": [
                {
                  "field_name": "docNo",
                  "operator": "=",
                  "value": "20250515001102"
                },
                {
                  "field_name": "date",
                  "operator": "=",
                  "value": "20090515"
                }
              ],
              "fields_operator": "AND"
            }
          ]
        }
      ]
    }
  }
}

在这个范例中，WHERE有二个字段，因此需要二个条件式。

因为有二个条件式，operator为这二个条件式之间的逻辑运算符，可以为AND或OR。此范例中使用AND。

同样的最外层的groups_operator为条件组之间的逻辑运算符。而fields_operator为此条件组内，条件式之间的逻辑运算符。
