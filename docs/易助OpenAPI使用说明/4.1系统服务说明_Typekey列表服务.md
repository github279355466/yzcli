# Typekey列表服务

> 所属章节: 系统服务说明

---

获取服务帮助信息，所有的字段属性。

服务接口名称命名规则：yz.oapi.TypeKeyList

#### 输入字段

| 标签 | 说明 |
| --- | --- |
|  |  |

#### 返回字段

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |
| typekeys | Typekey列表，包含所有可用TypeKey，在作业服务中需要使用 |
| name | Typekey调用名 |
| description | Typekey中文描述 |
| subservices | 子服务列表包含：query|查询，get|读取，create|新增，update|更新，delete|删除，approve|审核，disapprove|撤销审核 |
| name | 具体子服务 |

#### 示例场景

yz.oapi.TypeKeyList  搭配 帮助信息服务yz.oapi.{Typekey}.help，可以生成TPASC21所有功能，是一个动态的在线OpenAPI文档

## Request

### http-header

#### 示例

Content-Type: application/json; charset=utf-8
digi-service: {"name":"yz.oapi.typekeyList"}
digi-user-token: Ht7XKsAioLgYgf0+uL34Ar/uv1w315Edkvw29ExXYyA=(此值通过ERP作业 设置授权人员 TPASC19获取)

### http-body

#### 示例

{}
