# 刷新参数服务

> 所属章节: 系统服务说明

---

刷新账套参数和用户参数。

服务接口名称命名规则：yz.oapi.refresh

#### 输入字段

| 标签 | 说明 |
| --- | --- |
|  |  |

#### 返回字段

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |

#### 示例场景

当通过易助OpenAPI或者ERP更改了系统参数或者节点配置之后，需要通过此接口刷新参数缓存数据

## Request

### http-header

#### 示例

Content-Type: application/json; charset=utf-8
digi-service: {"name":"yz.oapi.refresh"}
digi-user-token: Ht7XKsAioLgYgf0+uL34Ar/uv1w315Edkvw29ExXYyA=(此值通过ERP作业 设置授权人员 TPASC19获取)

### http-body

#### 示例

{}

## Response

#### 示例

{
  "std_data": {
    "execution": {
      "code": "0",
      "sql_code": "",
      "description": "yz.oapi.refresh服务执行成功"
    },
    "parameter": {
      "result": {
        "success": [
          {
            "company": "DSBBeta"
          }
        ],
        "error": []
      }
    }
  }
}
