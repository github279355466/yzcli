# 审核服务

> 所属章节: 子服务说明

---

审核一笔/多笔资料或单据。

服务接口名称命名规则：yz.oapi.{Typekey}.approve

数据定位方式：业务主键。

#### 输入字段

| 标签 | 说明 |
| --- | --- |
| datakeys | 服务对应cdsMaster数据的主键字段 |

#### 返回字段

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |

#### 示例场景

审核生产入库单,单号:20250218000001  和  单号:20250403000011 详细信息。

## Request

### http-header

#### 示例

Content-Type: application/json; charset=utf-8
digi-service: {"name":"yz.oapi.wo.stock.in.approve"}
digi-user-token: Ht7XKsAioLgYgf0+uL34Ar/uv1w315Edkvw29ExXYyA=(此值通过ERP作业 设置授权人员 TPASC19获取)

### http-body

#### 示例

{
  "std_data": {
    "parameter": {
      "datakeys": [
        {
          "docNo": "20250218000001",
          "category": "84"
        },
        {
          "docNo": "20250403000011",
          "category": "84"
        }
      ]
    }
  }
}
