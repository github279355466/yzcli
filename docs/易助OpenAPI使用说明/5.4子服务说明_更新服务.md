# 更新服务

> 所属章节: 子服务说明

---

更新一笔/多笔资料或单据。接口服务更新的数据905字段以60000开始编码

服务接口名称命名规则：yz.oapi.{Typekey}.update

约束：

#### 输入字段

| 标签 | 说明 |
| --- | --- |
| cdsMaster | 服务对应cdsMaster数据的字段 |
| cdsDetail | 服务对应cdsDetail数据的字段,注意结构位置需要与列表中一致 |
| cdsDetail2 | 服务对应cdsDetail2数据的字段,注意结构位置需要与列表中一致 |

#### 返回字段

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |

#### 示例场景

获取生产入库单,单号:20250218000001  和  单号:20250403000011 详细信息。

## Request

### http-header

#### 示例

Content-Type: application/json; charset=utf-8
digi-service: {"name":"yz.oapi.sales.order.update"}
digi-user-token: Ht7XKsAioLgYgf0+uL34Ar/uv1w315Edkvw29ExXYyA=(此值通过ERP作业 设置授权人员 TPASC19获取)

### http-body

#### 示例

{
  "std_data": {
    "parameter": {
      "cdsMaster": [
        {
          "docNo": "20250408000001",
          "nature": "T2",
          "customer": "000002",
          "cdsDetail": [
            {
              "docNo": "20250408000001",
              "seq": "001",
              "item_no": "001B",
              "qty": "5"
            }
          ]
        }
      ]
    }
  }
}

## Response

#### 示例

{
  "std_data": {
    "execution": {
      "code": "0",
      "sql_code": "",
      "description": "yz.oapi.sales.order.update服务执行成功"
    },
    "parameter": {
      "result": {
        "success": [
          {
            "keyVaule": {
              "docNo": "20250408000001"
            }
          }
        ],
        "error": []
      }
    }
  }
}
