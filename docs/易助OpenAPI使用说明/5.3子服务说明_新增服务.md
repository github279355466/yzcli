# 新增服务

> 所属章节: 子服务说明

---

新增一笔/多笔资料或单据。接口服务新增的数据905字段以50000开始编码

如单据设置自动审核，保存成功后会自动进行审核。

服务接口名称命名规则：yz.oapi.{Typekey}.create

约束：新增资料，必须提供数据的业务主键，以及不可空白字段。

如果规则是自动生成单号等系统自动进行默认赋值，无需提供。因此可以参考手动做单填写项来组Json数据

长度超出字符型字段最大长度的数据，会自动截取

#### 输入字段

| 标签 | 说明 |
| --- | --- |
| cdsMaster | 服务对应cdsMaster数据的字段 |
| cdsDetail | 服务对应cdsDetail数据的字段,注意结构位置需要与列表中一致 |
| cdsDetail2 | 服务对应cdsDetail2数据的字段,注意结构位置需要与列表中一致 |

#### 返回字段

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |

#### 示例场景

新增订单数据。
