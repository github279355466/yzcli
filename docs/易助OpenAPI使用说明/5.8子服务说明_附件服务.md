# 附件服务

> 所属章节: 子服务说明

添加附件文件获取位置：

用于配置添加附件接口attachmentAdd，添加附件时需要从指定目录去寻找文件

获取附件临时文件保存位置：

用于配置获取附件接口attachmentGet，会把附件文件保存至指定目录下。

如果临时文件已经存在且后端没有更新附件，接口会直接复用，省去重新生成的步骤。

获取附件文件地址前缀：

如果为空则直接输出文件的保存位置。需要确保第三方系统可以正常访问到此文件夹。

配置的文件地址前缀可以是IP 共享文件夹的路径，比如： \\172.16.6.22\TPA。需要确保第三方系统可以正常访问到windows的共享文件夹。

如果在服务器安装了Tomcat，则可以直接指定到Webapps下，可以直接浏览器访问。必须配合勾选上URL勾选框使用，这样才会把文件夹路径的 \  转换成 URL的 /

# 附件查询服务

查询单据的附件列表。

服务接口名称命名规则：yz.oapi.{Typekey}.attachmentQuery

数据定位方式：业务主键。

#### 输入字段

| 标签 | 说明 |
| --- | --- |
| datakeys | 服务对应cdsMaster数据的主键字段 |

#### 返回字段

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |

#### 示例场景

查询订单附件,单号:20251104000002 和  单号:20251104000005 详细信息列表。

## Request

### http-header

#### 示例

Content-Type: application/json; charset=utf-8
digi-service: {"name":"yz.oapi.sales.order.attachmentQuery"}
digi-user-token: Ht7XKsAioLgYgf0+uL34Ar/uv1w315Edkvw29ExXYyA=(此值通过ERP作业 设置授权人员 TPASC19获取)

### http-body

#### 示例

{
  "std_data": {
    "parameter": {
      "datakeys": [
        {
          "docNo": "20251104000002"

          "docNo": "20251104000005"

    }
  }
