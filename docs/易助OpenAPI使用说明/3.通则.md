# 通则

---

此章节主要用于介绍产品集成相关的通用规则。如集成入口，Request/Reponse参数标签说明等。

## 集成协议规范

| URL | http://{IP}:{端口号}     如：http://172.16.6.22:8103 |
| --- | --- |
| 格式 | JSON |
| 请求方式 | POST |
| 编码 | UTF-8 |

## 子服务清单

| 子服务 | 描述 | 接口详细文档 |
| --- | --- | --- |
| query | 查询 | 子服务说明->查询服务 |
| get | 读取 | 子服务说明->读取服务 |
| create | 新增 | 子服务说明->新增服务 |
| update | 更新 | 子服务说明->更新服务 |
| delete | 删除 | 子服务说明->删除服务 |
| approve | 审核 | 子服务说明->审核服务 |
| disapprove | 撤销审核 | 子服务说明->撤审服务 |

## Request 说明

### http-header

#### 报头

| 参数名 | 参数值 | 是否必须 | 描述 |
| --- | --- | --- | --- |
| Content-Type | application/json; charset=utf-8 | √ | 内容格式, 固定为: application/json; charset=utf-8 |
| digi-service | {"name":"接口名"} | √ | 接口名, 可以通过yz.oapi.TypeKeyList服务获取或者易助ERP中的TPASC21查询, 例如: yz.oapi.item.query yz.oapi. 为固定前缀 item 为TypeKey query 为子服务 |
| digi-user-token | 用户令牌 | √ | 调用接口时根据此令牌  定位账套 进行身份认证校验权限 |

#### 示例

Content-Type: application/json; charset=utf-8
digi-service: {"name":"yz.oapi.warehouse.get"}
digi-user-token: Ht7XKsAioLgYgf0+uL34Ar/uv1w315Edkvw2 (此值通过ERP作业 设置授权人员 TPASC19获取)

### http-body

#### 报文

| 标签 | 是否必要 | 描述 |
| --- | --- | --- |
| std_data | √ | 数据固定标签 |
| parameter | √ | parameter下为各个API的输入参数，具体可查看各API的操作说明。 |

#### 示例

{
 "std_data": {
  "parameter": {
   "datakeys": [
    {
     "warehouse_no": "023153"
    },{
     "warehouse_no": "zzzzz"
    }]
  }
 }
}

## Response说明

#### 返回

| 标签 | 描述 |
| --- | --- |
| std_data | 固定标签，返回数据的根标签 |
| execution | 固定标签，用于服务调用的状态 |
|  | 状态码：
            0: 表示服务调用成功
           -1: 表示服务调用失败 -0: 表示服务调用部分成功 批量调用时，成功结果返回在result. success ,失败结果返回在result.error |
| description | 状态码为-1时，表示具体异常信息 |
| sql_code | 系统预留字段 |
| parameter | API服务返回内容 |
| result | 其中包含success和error标签，分别存放成功/失败的数据处理状态。 |

#### 示例

执行成功

{
 "std_data": {
  "execution": {
   "code": "0",
   "sql_code": "",
   "description": "yz.oapi.warehouse.get服务执行成功"
  },
  "parameter": {
   "result": {
    "success": [
     {
      "cdsMaster": [
       {
        "warehouse_no": "023153",
        "warehouse_nature": "2.现场仓",
        "warehouse_name": "32415646",
        "include_demand_calculation": "F",
       }]
     }],
    "error": []
   }
  }
 }
}

执行失败

{

   "code": "-1",

 }
}
