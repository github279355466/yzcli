# OpenClaw 平台：易助 ERP Agent 防御手册集成指南

本指南旨在指导你如何将《易助 ERP Agent 幻觉与参数遗漏防御手册》中的策略深度集成到 OpenClaw 平台，构建一个稳健、准确的 ERP 自动化助手。

---

## 1. 核心集成点：System Prompt (系统提示词)

在 OpenClaw 的 Agent 配置中，System Prompt 是防御幻觉的第一道防线。

### 操作步骤：
1. 进入 OpenClaw **Agent 设置** -> **System Prompt**。
2. 插入以下“防御性指令块”：

```markdown
## ⚠️ 严苛执行准则 (Strict Execution Rules)
- **禁止猜测编码**：严禁根据拼音或简称发明客户/物料编码。若用户未提供精确编码，必须先调用 `fastquery` 接口查询。
- **字段白名单**：仅允许使用 `cli_contract.md` 中定义的节点名称。严禁生成任何自定义或猜测的 JSON 键名。
- **结构完整性**：创建订单时，必须同时包含 `cdsMaster` 和 `cdsDetail`。单身行的 `docNo` 必须与单头完全一致。
- **数据标准化**：日期必须转换为 `YYYYMMDD`，数值必须为纯数字（Float/Int），禁止携带单位。
- **信息缺失处理**：若必填项（如单价、客户）缺失，必须暂停并询问用户，严禁设为默认值或 0。
```

---

## 2. 流程集成：先查后写 (Read-Before-Write)

为了防止 Agent 错误地使用客户名称代替编码，需要在 OpenClaw 中配置“强制查询流”。

### 配置逻辑：
在 OpenClaw 的 **技能描述 (Tool Description)** 中，为 `create` 动作增加前置说明：
> “在调用此工具创建单据前，请检查输入参数。如果参数中包含的是中文名称而非字母数字组成的编码，请先调用 `fastquery` 工具获取对应的 `cust` 或 `item_no`。”

---

## 3. 结构化约束：JSON Schema 校验

如果 OpenClaw 支持参数 Schema 校验，请务必开启。

### 校验重点：
- **docNo**: 正则表达式约束（如 `^[A-Z0-9-]{1,14}$`）。
- **date**: 长度固定为 8 位，且为数字。
- **cdsDetail**: 必须为非空数组。

---

## 4. 调试与负面测试 (Negative Testing)

集成后，请在 OpenClaw 调试控制台进行以下“恶意”测试，确保防御生效：

| 测试场景 | 预期 Agent 反应 |
| --- | --- |
| **模糊输入**：“给张三下个单” | Agent 不应直接下单，而应先调用查询接口或询问张三的编码。 |
| **缺失必填项**：“录入一张订单，买 10 个螺丝” | Agent 应询问：“请问单价是多少？对应的客户是谁？” |
| **格式干扰**：“日期是 2026年4月29日” | Agent 组装的 JSON 中日期应为 `20260429`。 |
| **非法字段**：“查询订单的物流状态” | Agent 应回复该字段未定义，而不是编造一个 `shipping_status`。 |

---

## 5. 持续优化

- **日志审计**：定期查看 OpenClaw 的 `Tool Call` 日志。如果发现 Agent 绕过了防御逻辑，请在 System Prompt 中使用更强烈的语气（如“必须”、“严禁”、“否则会导致系统崩溃”）。
- **知识库同步**：确保 `cli_contract.md` 始终是最新的，Agent 才能有据可查。

---
*通过以上配置，你的 OpenClaw Agent 将具备极高的业务确定性，能够安全地处理真实的 ERP 生产数据。*
